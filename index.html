前言

　　本文主要记录 管理决策系统（大数据项目） 开发和维护相关的一些注意事项.

项目简介

　　决策管理项目为内嵌在 跟单宝app 内的二级子页面

准备工作

　　项目环境：

"node": ">= 4.0.0"

"npm": ">= 3.0.0"

"webpack": "^3.6.0"

目录结构

touna-data-static----------------------------决策管理项目
  ├─build------------------------------打包配置
  ├─config-----------------------------打包环境配置
  ├─dist-------------------------------打包后生成的目录结构
  ├─src--------------------------------项目源码
  │  ├─admin--------------------管理员模块
  │  ├─api----------------------公共请求
  │  ├─assets-------------------公共静态资源
  │  ├─axios--------------------第三方请求模块
  │  ├─components---------------项目公共组件
  │  ├─router-------------------项目路由
  │  ├─scss---------------------项目内组件公共样式
  │  ├─store--------------------项目状态管理
  │  ├─utils--------------------项目公共工具类
  │  ├─views--------------------项目视图层
  │  ├─app.vue------------------项目首页页面
  │  ├─config.js----------------项目打包环境配置
  │  └─main.js------------------项目路由入口
  └─static-----------------------------项目基准样式&组件引用样式

编译&启动

　　参见根目录下 package.json 中配置

npm run start  本地启动，默认代码热更新，默认端口8080，启动成功后，打开浏览器访问localhost:8080
npm run build  编译本地代码并压缩混淆打包成静态资源文件，拷贝至对应服务器上指定路径即可完成部署
npm run build --report 编译本地代码并输出报告

代码中的主体实现逻辑在哪儿？

1.topbar组件中的监听(watch钩子)

watch钩子的触发时机：

  • 首次进入项目首页

  • 每次更改下拉框选中值

watch: {
    busnissSelect(nVal) {
    let busnissListName = this.busniessList.map(curVal => curVal.name);
    let index = busnissListName.indexOf(nVal.name);

    this.$store.commit('areaList', this.busniessList[index].bigAreas || [{code: null, name: "大区", provinceAreas: null}]);
    this.$store.commit('areaSelect', this.areaList[0].code ? this.areaList[0] : {code: null, name: "大区", provinceAreas: null});
  }

　 以busnissSelect为例，逻辑有二：

  • 根据val在当前list中的序号，初始化下一级list
  • 初始化下一级list的选中值

需要注意，此监听会同时影响 areaSelect 、 provinceSelect 和 citySelect ，在最后的 citySelect 中，调用了 changeLevel 方法来改变全局变量 level 值(标志量：下拉框选中的值对应的权限级别)

  changeLevel() {
    let bsnsCode = this.busnissSelect ? this.busnissSelect.code : null;    //获取事业部选中值
    let areaCode = this.areaSelect ? this.areaSelect.code : null;            //获取大区选中值
    let proCode = this.provinceSelect ? this.provinceSelect.code : null;  //获取省份选中值
    let cityCode = this.citySelect ? this.citySelect.code : null;            //获取公司选中值

    //改变level
    if (bsnsCode && areaCode && proCode && cityCode) {
      this.$store.commit('level', 'Site');
    }else if (bsnsCode && areaCode && proCode) {
      this.$store.commit('level', 'ProvinceArea');
    }else if (bsnsCode && areaCode) {
      this.$store.commit('level', 'BigArea');
    }else if (bsnsCode) {
      this.$store.commit('level', 'BsnsUnit');
    }else if (!bsnsCode) {
      this.$store.commit('level', 'All');
    }
}
2.index.vue中的初始化操作（mounted钩子）

mounted钩子触发时机：

  • 从子页面返回至首页

    mounted() {
  if (this.areaSelect !== null) {//后退到首页时，进入mounted，初始化一次，不能放到created中，因为dom还没初始化完成，水印加不上
    this.init();
    this.initRank();
  }
}

　　initRank 必须在dom加载完成后执行来添加水印，而将 initRank 功能单独从 init 中提取出来，是因为二期新增的需求(切换下拉框公司时，排名模块的展示值要相应改变)。这项改变依赖于 level 和 levelRange， 故提取出来另做判断。
